这些扩展模块并非各自为战的孤岛，而是围绕“练琴”这个核心业务，构成了一个**从“输入 $\rightarrow$ 计划 $\rightarrow$ 执行 $\rightarrow$ 记录 $\rightarrow$ 组织 $\rightarrow$ 激励”的完整生命周期闭环**。这会让你的系统从一个“工具”升维成一个真正的“生态”。

# 🎼《数字化乐谱资源管理与协作系统》产品需求文档 (PRD v4.0)

## 一、 项目概述 (Project Overview)
*   **项目名称**：数字化乐谱资源管理与协作系统
*   **产品别名**：LyraScore 云端谱仓
*   **核心定位**：一个为现代乐器学习者设计的、集**乐谱资产管理、系统化练习规划、游戏化激励、私域琴友互动**于一体的全生命周期云端协作平台。本项目旨在通过深度挖掘“练琴”业务流程，解决传统乐谱管理混乱、练习过程缺乏记录与反馈、琴友间交流不便的核心痛点。

## 二、 用户角色与权限 (User Roles & Permissions)
1.  **琴友（User）**：系统的核心使用者。在不同场景下拥有不同权限：
    *   **作为主人（Owner）**：在自己的谱仓、练习计划、歌单中拥有完全的增删改查权限。
    *   **作为访客（Friend）**：在好友的谱仓中拥有只读、快捷私信和点歌的权限。
2.  **系统管理员（Admin）**：后台系统的维护者。负责用户账号治理、数据大盘统计，以及对系统内的**“字典数据”（如快捷短语、徽章解锁条件）**进行配置。

## 三、 核心功能模块清单 (Functional Features List)

### 模块一：谱仓资产核心模块（基础数据层）
| 功能名称 | 功能描述 | 数据库设计亮点提示 |
| :--- | :--- | :--- |
| **乐谱资产入库** | 上传吉他谱图片，填写歌曲名、歌手、调式、BPM等元数据，并撰写个人练习备忘录。 | `1:N` 关系 (用户:乐谱) |
| **长图拼接分享** | **（前端特色）** 一键将曲谱图片与备忘录拼接成一张长图，供用户保存至本地相册分享。 | 降低后端I/O与带宽压力 |

### 模块二：练习计划与日志模块（核心业务流）
| 功能名称        | 功能描述                                   | 数据库设计亮点提示                                     |
| :---------- | :------------------------------------- | :-------------------------------------------- |
| **创建练习计划**  | 创建有时限的练习计划（如“本周爬格子专项”），并从谱仓中关联多首乐谱。    | `t_practice_plan` 表 (1:N), `t_plan_items` 关联表 |
| **练习计时与日志** | 针对计划中的某首乐谱，启动计时器记录练习时长。结束后可撰写本次练习日志。   | `t_practice_log` 表 (1:N)                      |
| **练习数据统计**  | 自动聚合统计用户总练习时长、周/月练习曲线、最常练习曲目等，并以图表可视化。 | **SQL 聚合查询** (`GROUP BY`, `SUM`)              |

### 模块三：歌单（Setlist）管理模块（数据组织层）
| 功能名称 | 功能描述 | 数据库设计亮点提示 |
| :--- | :--- | :--- |
| **创建与管理歌单** | 创建多个不同主题的歌单（如“演出歌单”、“民谣精选”）。 | `t_setlist` 表 (1:N) |
| **曲目收藏与排序** | 从自己或好友的谱仓中收藏乐谱进歌单，并可自定义歌单内曲目的顺序。 | **`t_setlist_items` (N:M 多对多关联表)** |

### 模块四：琴友社交与互动模块（关系与交互层）
| 功能名称         | 功能描述                                                                     | 数据库设计亮点提示                      |
| :----------- | :----------------------------------------------------------------------- | :----------------------------- |
| **琴友关系链**    | 通过用户名搜索、发送好友请求、同意/拒绝、解除好友关系。                                             | **`t_friendship` (自引用多对多关系)**  |
| **AI 智能点歌台** | **（创新亮点）** 用户自由输入想听的歌曲，**后端调用AI API进行内容清洗与识别**，将干净的歌名与歌手信息存入点歌列表。杜绝垃圾信息。 | **AI 接口调用**，`t_song_request` 表 |
| **双态点歌队列**   | 点歌消息自动双色高亮区分“谱仓内已有”与“谱仓内暂无”，便于接收方处理。                                     | **多表联合查询 (JOIN)**              |

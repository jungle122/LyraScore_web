# 🎼《数字化乐谱资源管理与协作系统》产品需求文档 (PRD v5.0)

> v5.0 与代码实际状态对齐：删除未实现的「长图拼接 / 双态点歌队列 / 管理员后台」三项；徽章方案从「数据库触发器」改为「Spring `@EventListener` 事件驱动」；新增「小工具集 / AI 练习周报 / 谱仓增强 / 找谱搜索」四块。

## 一、项目概述 (Project Overview)

*   **项目名称**：数字化乐谱资源管理与协作系统
*   **产品别名**：LyraScore 云端谱仓
*   **核心定位**：为现代乐器学习者设计的、集**乐谱资产管理 / 系统化练习规划 / 游戏化激励 / 私域琴友互动 / AI 智能助手 / 实用音乐工具**于一体的全栈协作平台。深度挖掘「练琴」业务流程，解决传统乐谱管理混乱、练习过程缺乏记录与反馈、琴友间交流不便的核心痛点。

## 二、用户角色与权限

1.  **琴友（User）**：系统的核心使用者
    *   作为主人（Owner）：在自己的谱仓、练习计划、歌单、日志中拥有完全的 CRUD 权限
    *   作为访客（Friend）：可向好友点歌、添加好友
2.  **管理员（Admin）**：`t_user.role = 'admin'`，目前仅作为权限位预留，**管理员后台 UI 暂未实现**

## 三、核心功能模块清单

### 模块一：谱仓资产核心模块（基础数据层）✅

| 功能名称 | 功能描述 | 数据库 / 技术亮点 |
|---|---|---|
| **乐谱资产入库** | 上传吉他谱图片（**支持多张**，最多 12 张），填写歌曲名、歌手、调式、变调夹、BPM、乐器、风格、备忘录 | `1:N`（用户:乐谱）；image_url 用 `VARCHAR(2000)` 逗号分隔多图 |
| **元数据编辑** | 已上传乐谱可编辑全部元数据（图片不可改，需删除重传） | `PUT /scores/{id}` 部分字段更新 |
| **练习状态切换** | 标记乐谱为「未开始 / 正在练 / 已练完」 | `practice_status` 字段 + `PUT /scores/{id}/status` |
| **筛选与搜索** | 谱仓页支持按乐器/风格/状态筛选 + 关键字模糊搜索 + 三种排序（最新/最早/A-Z） | 前端 computed 实时过滤 |
| **阅读工具条** | 详情页「1 页 / 2 页显示」切换、「大图预览」（el-image-viewer）、「全屏显示」（Fullscreen API） | CSS Grid + `requestFullscreen` |
| **删除策略** | 删除乐谱时级联清理歌单/计划关联，但保留练习日志 | 外键 CASCADE（关联表）+ RESTRICT（日志） |

### 模块二：练习计划与日志模块（核心业务流）✅

| 功能名称 | 功能描述 | 数据库 / 技术亮点 |
|---|---|---|
| **创建练习计划** | 创建有时限的练习计划（如「本周指弹专项」），并从谱仓中关联多首乐谱，可标记完成状态 | `t_practice_plan`（1:N）+ `t_plan_items`（N:M 关联表） |
| **记录练习日志** | 针对某首乐谱写练习日志：时长、达到的 BPM、心得 | `t_practice_log`（1:N） |
| **聚合统计** | 自动统计：累计/本周分钟、按乐谱分组累计、近 7 日按日分组——所有数据可视化为 tile + 表格 | **SQL `GROUP BY`、`SUM`、`DATE()` 聚合**（教学目标） |
| **AI 练习周报** | 一键生成 100-200 字 AI 教练点评：累计时长亮点 + 最爱曲目分析 + 改进建议 + 鼓励 | 阿里云百炼 qwen-turbo + 失败本地降级 |

### 模块三：歌单（Setlist）管理模块（数据组织层）✅

| 功能名称 | 功能描述 | 数据库 / 技术亮点 |
|---|---|---|
| **创建与管理歌单** | 创建多个不同主题的歌单（如「演出歌单」「民谣精选」） | `t_setlist`（1:N） |
| **曲目收藏** | 从自己谱仓/公开乐谱中收藏进歌单 | **`t_setlist_items`（N:M 多对多关联表）** + `uk_setlist_items_pair` 唯一键防重复收藏 |

### 模块四：琴友社交与互动模块（关系与交互层）✅

| 功能名称 | 功能描述 | 数据库 / 技术亮点 |
|---|---|---|
| **琴友关系链** | 用户名搜索发起申请、接受/拒绝、解除关系；列表区分「我的好友 / 待处理 / 添加好友」三 tab | **`t_friendship`（自引用 N:M）**：CASE 表达式动态选取「对端用户」 |
| **AI 智能点歌台** | 用户自由输入想听的歌，**后端调阿里云百炼 qwen-turbo 清洗**为 `(歌名, 歌手)`；system prompt 内置艺名归一化（霉霉→Taylor Swift / 周董→周杰伦 等）；用户对结果不满可点「重新清洗」 | **AI 接口调用**，`t_song_request` 表，失败安全（status=2 待人工审核） |
| **快捷短语点歌** | 点歌时可附带预设快捷短语（如「大佬教教我」）作为留言 | `t_dictionary`（dict_type='quick_dm'）+ JOIN 显示 |

### 模块五：成就徽章系统（游戏化激励层）✅

| 功能名称 | 功能描述 | 数据库 / 技术亮点 |
|---|---|---|
| **徽章定义** | 系统预置 17 张徽章，覆盖 7 种触发条件类型（first_log / total_minutes / log_count / score_count / plan_count / setlist_count / friend_count / song_request_count） | `t_badge` 字典表 |
| **自动解锁** | 用户行为（写日志、上传谱、加好友、点歌、建计划/歌单）发生后**自动评估并发牌**，无需手动触发 | **Spring `@EventListener` 事件驱动**：`PracticeLogCreatedEvent` 监听 + 关键 service 直调 `BadgeService.evaluateAndAward`；`INSERT IGNORE` + `uk_user_badge_pair` 实现幂等（替代早期方案中的 DB 触发器） |
| **进度可视化** | 徽章页未解锁徽章显示进度条（如「已练 45 / 60 分钟」） | LEFT JOIN 查询 + 前端进度条 |

### 模块六：实用音乐工具（应用闭环）✅

| 功能名称 | 功能描述 | 技术亮点 |
|---|---|---|
| **吉他定音器** | 5 种调弦预设（Standard / Drop D / Open D / Open G / DADGAD）+ 整体降调 0~-6 半音；指板俯视图 6 弦交互 | **Web Audio API**：预加载 6 弦 mp3 → `BufferSource.playbackRate = 2^(semitones/12)` 实现变调 |
| **节拍器** | 30-250 BPM + 4 种拍号（4/4 / 3/4 / 2/4 / 6/8）；强拍音色更亮 | `setInterval` 定时 + `playbackRate=1.5` 制造重音 |
| **找谱搜索** | 一个搜索框 + 3 个吉他谱站（91pu 繁体 / 弹琴吧 / 不休吉他）；输入简体即可，**繁体站自动转换后再搜**；支持「一键多站搜」 | **opencc-js** 简繁词典 + `window.open()` 多标签 |

## 五、技术亮点速览（用于报告 + 答辩）

1. **三种数据关系**：1:N、N:M、**自引用 N:M**（用 `CASE WHEN` 动态选对端）
2. **SQL 聚合**：`GROUP BY` + `SUM` + `COUNT` + `DATE()` 函数
3. **JOIN 教学**：计划详情页用 INNER JOIN 拉乐谱字段，避免 N+1 查询
4. **外键策略分级**：CASCADE（纯关联表）vs RESTRICT（历史日志）
5. **事件驱动**：`@EventListener` 解耦业务主流程和徽章评估
6. **手写 JWT 拦截器**：不依赖 Spring Security，理解原理
7. **AI 双场景集成 + 失败降级**：点歌清洗 + 练习周报，AI 失败时业务不中断
8. **本地存储策略**：图片落本地磁盘 + DB 存相对路径；不引入对象存储/CDN

### 🗺️《LyraScore 云端谱仓》全生命周期开发计划

整个项目分为 **6 个大阶段**，采用「演化模型 + 垂直切片」，确保每做完一小步系统都是能跑通的。

#### 阶段一：产品定义与纸上原型（需求期）
*   **任务 1**：确定项目核心概念与功能范围。✅ 已完成（[[PRD]] v5.0）
*   **任务 2**：手绘/AI 生成核心界面草图，确定视觉方向。✅ 已完成

#### 阶段二：数据模型构建（架构期）
*   **任务 3**：梳理系统核心实体与关系，绘制 **[[E-R 图]]**。✅ 已完成
*   **任务 4**：将 E-R 图转化为详细的数据库表结构设计（字段名、数据类型、主外键、索引、外键策略 RESTRICT/CASCADE）。✅ 已完成

#### 阶段三：基建与第一刀（跑通环境）
*   **任务 5**：初始化项目骨架（Spring Boot 3.2 + Vue 3 + Element Plus）。✅ 已完成
*   **任务 6**：配置本地 MySQL 8.0 + MyBatis（XML mapper）连接。✅ 已完成
*   **任务 7**：**垂直切片 1（用户闭环）**：注册/登录 + JWT 拦截器 + Pinia 用户态 + 路由守卫 + 注册页。✅ 已完成

#### 阶段四：核心业务迭代（剥洋葱式开发）
*   **任务 8**：**垂直切片 2（谱仓闭环）**：乐谱上传（**多图**）、本地图片存储、CRUD、详情查看。
    后期增强：练习状态切换、乐器/风格筛选、关键字搜索、排序、元数据编辑、阅读工具条（1页/2页/大图预览/全屏）。✅ 已完成
*   **任务 9**：**垂直切片 3（计划+日志闭环）**：练习计划 N:M 关联乐谱、写日志、聚合统计（**GROUP BY 教学点**）、`PracticeLogCreatedEvent` 事件埋点。✅ 已完成
*   **任务 10**：**垂直切片 4（歌单 + 社交）**：歌单 N:M 收藏、好友关系（**自引用 N:M** + `CASE` 表达式查询「对端用户」）、关联表 CASCADE 删除策略。✅ 已完成

#### 阶段五：高阶特性注入（AI 与亮点）
*   **任务 11**：集成阿里云百炼 **qwen-turbo**（OpenAI 兼容端点），实现「AI 智能点歌信息清洗」+「重新清洗」+ 失败降级（status=2 待人工审核）。✅ 已完成
*   **任务 12**：用 Spring **`@EventListener`** 订阅 `PracticeLogCreatedEvent`，自动评估并授予徽章（**事件驱动架构**，替代 PRD 早期提到的 DB 触发器方案）。`INSERT IGNORE` 配合 `uk_user_badge_pair` 唯一键实现幂等。✅ 已完成（17 张徽章 / 7 种触发条件）

#### 阶段五补充：垂直工具与体验增强
*   **小工具**：吉他定音器（5 种调弦预设 + Web Audio API + `playbackRate = 2^(semitones/12)` 变调）、节拍器（30-250 BPM + 强弱拍变调）、找谱搜索（opencc-js 简繁转换 + 多站快搜）。✅ 已完成
*   **第二个 AI 场景**：AI 练习总结（`/logs/report` 接口，把 stats JSON 喂给 qwen-turbo 生成 120-200 字教练点评，失败本地降级文案）。✅ 已完成
*   **管理员后台**：数据大盘（跨表 COUNT/SUM 聚合）+ 用户管理（冻结/解冻）+ 字典管理（CRUD）；后端 `@RequireRole("admin")` 注解 + JwtInterceptor 反射拦截 + 前端路由守卫 + 侧边栏按角色切换视图。**首次实战使用 RBAC 注解机制**。✅ 已完成

#### 阶段六：课设报告与答辩准备（收尾期）
*   **任务 13**：将 PRD、E-R 图、数据库设计、核心代码截图按学校模板填入 Word 文档。
*   **任务 14**：本地浏览器无痕模式双开账号，录制 5-10 分钟演示视频。

### 🗺️ 《LyraScore 云端谱仓》全生命周期开发计划

整个项目分为 **6个大阶段**。我们采用“演化模型 + 垂直切片”，确保你每做完一小步，系统都是能跑通的。

#### 阶段一：产品定义与纸上原型（需求期）
*   **任务 1**：确定项目核心概念与功能范围。（✅ **已完成：[[PRD]] v4.0**）
*   **任务 2**：手绘/使用 Google AI Studio 生成核心界面草图，确定视觉方向。（✅ **已完成/你随时可做**）

#### 阶段二：数据模型构建（架构期）
*   **任务 3**：梳理系统核心实体与关系，绘制 **[[E-R 图]]**（实体关系图）。
*   **任务 4**：将 E-R 图转化为详细的 **数据库表结构设计**（确定字段名、数据类型、主外键）。

#### 阶段三：基建与第一刀（跑通环境）
*   **任务 5**：让 Copilot Agent 帮你初始化项目骨架（比如 Spring Boot + Vue 3 的空壳）。
*   **任务 6**：配置本地 MySQL 数据库连接。
*   **任务 7**：**垂直切片 1（用户闭环）**：完成用户注册/登录的数据库表创建、后端接口编写、前端页面联调。确保系统能“活”起来。

#### 阶段四：核心业务迭代（剥洋葱式开发）
*   **任务 8**：**垂直切片 2（谱仓闭环）**：实现乐谱上传、图片存储、列表展示。
*   **任务 9**：**垂直切片 3（计划闭环）**：实现练习计划的创建与日志记录。
*   **任务 10**：**垂直切片 4（歌单与社交）**：实现多对多关系的歌单收藏、好友关系模拟。

#### 阶段五：高阶特性注入（AI 与亮点）
*   **任务 11**：集成大模型 API（如调用 Gemini 3.1 或其他免税额度的 API），实现“AI 智能点歌信息清洗”功能。
*   **任务 12**：配置简单的数据库触发器或后端逻辑，实现“成就徽章”自动发放。

#### 阶段六：课设报告与答辩准备（收尾期）
*   **任务 13**：把前五个阶段产出的 PRD、E-R图、数据库设计表、核心代码截图，按学校要求的模板填入 Word 文档。
*   **任务 14**：在本地浏览器使用“无痕模式”双开账号，录制一段完美的演示视频。

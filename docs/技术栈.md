### 🛠️《LyraScore》全栈技术方案

> 前后端完全分离 / 本地部署 / 课程设计场景下尽量精简依赖

---

#### 1. 🎨 前端 (Frontend)

| 类别 | 技术 / 库 | 版本 | 选型理由 |
|---|---|---|---|
| 核心框架 | **Vue 3** | 3.x（Composition API） | 课程指定方向；响应式 + SFC 单文件组件，开发体验现代 |
| UI 组件库 | **Element Plus** | latest | 后台体面风格，Form / Table / Dialog / Card / Slider 一应俱全 |
| 构建工具 | **Vite** | 5.x | 冷启动 < 1s，HMR 极快，原生 ES Modules |
| 状态管理 | **Pinia** | 2.x | Vue 3 官方推荐；user store 统一管理 token + userInfo |
| 路由 | **Vue Router 4** | 4.x | `meta.requiresAuth` / `meta.guestOnly` 路由守卫 |
| HTTP 客户端 | **axios** | 1.x | 请求/响应拦截器：自动加 `Authorization: Bearer <token>`、剥 R 壳、401 自动 logout |
| 简繁转换 | **opencc-js** | latest | 找谱搜索功能用 `s2twp`（简体→台湾正体含词组词典） |
| 音频 API | **Web Audio API**（浏览器原生） | — | 定音器/节拍器：`AudioContext` + `decodeAudioData` + `BufferSource.playbackRate` 实现变调 |
| 图标 | **@element-plus/icons-vue** | latest | 统一替代 emoji，提升专业感 |
| 包管理 | npm + 国内镜像 `registry.npmmirror.com` | — | `.npmrc` 加速依赖下载 |

#### 2. ⚙️ 后端 (Backend)

| 类别 | 技术 / 库 | 版本 | 选型理由 |
|---|---|---|---|
| JDK | **OpenJDK** | 17 LTS | Spring Boot 3 最低要求 |
| Web 框架 | **Spring Boot** | 3.2.x | 全家桶；自动配置 + 内嵌 Tomcat |
| ORM | **MyBatis** | 3.0.x（无 MyBatis-Plus） | 课程目标是写 SQL；保留 XML mapper 教学性 |
| 安全 | **手写 JWT 拦截器**（不引 Spring Security） | jjwt 0.12 | 课程要求理解机制：`HandlerInterceptor` + `ThreadLocal` 存上下文 |
| 密码哈希 | **jBCrypt** | 0.4 | BCrypt 自带 salt，比 MD5/SHA 安全 |
| 数据校验 | Jakarta Validation | — | `@NotBlank` / `@Size` / `@Min` / `@Max` 注解校验 DTO |
| 工具库 | **Lombok** | latest | `@Data` / `@RequiredArgsConstructor` 减少样板 |
| HTTP 客户端 | **Spring RestClient** | Spring 6.1+ 内置 | 调用阿里云百炼 AI 接口，无需引入额外库 |
| JSON | **Jackson** | Spring Boot 默认 | RestController 自动序列化 + AI 响应解析 |
| 事件机制 | **Spring `@EventListener`** | 内置 | 替代 DB 触发器：`PracticeLogCreatedEvent` 解耦徽章发放逻辑 |

#### 3. 💾 数据库 (Database)

| 类别 | 技术 | 配置 | 说明 |
|---|---|---|---|
| RDBMS | **MySQL 8.0** | utf8mb4 + utf8mb4_0900_ai_ci | 本地安装；与 ERD 完全 1:1 落地 |
| 表数量 | **12 张** | 全部 `t_` 前缀 | t_user / t_score / t_friendship / t_song_request / t_practice_plan / t_plan_items / t_practice_log / t_setlist / t_setlist_items / t_dictionary / t_badge / t_user_badge |
| 关系类型 | **三种** | — | 1:N（用户→乐谱）/ N:M via 关联表（歌单-乐谱、计划-乐谱、用户-徽章）/ **自引用 N:M**（好友关系 user_id_1 + user_id_2） |
| 外键策略 | RESTRICT / CASCADE 混用 | — | 关联表（plan_items / setlist_items）→ CASCADE；练习日志 → RESTRICT（历史不可丢） |
| 索引 | 复合索引 + 唯一键 | `idx_score_user_created`、`uk_friendship_pair`、`uk_user_badge_pair` 等 | 性能 + 业务唯一性约束 |

#### 5. 🚀 运行环境与配置

- **平台**：Windows 10/11 + PowerShell（项目命令以 PS 为准）
- **环境变量**（写进 `$PROFILE`，**不进 git**）：
  - `DB_PASSWORD`：MySQL root 密码
  - `JWT_SECRET`：JWT 签名密钥
  - `DASHSCOPE_API_KEY`：阿里云 AI key
  - `AI_ENABLED`：`true` / `false`，关闭时所有 AI 调用降级
- **配置文件**：
  - 后端：`application.yml`（公共）+ `application-dev.yml`（开发环境，从环境变量读密钥）
  - 前端：Vite proxy 配置 `/api` 和 `/uploads` 都代理到后端 8080
- **构建工具**：
  - 后端：Maven（Aliyun 镜像加速）
  - 前端：Vite + npm（npmmirror 镜像）
- **图片存储**：本地磁盘 `backend/uploads/scores/` + UUID 命名；DB 仅存相对路径（多图逗号分隔）；后端 `WebMvcConfigurer.addResourceHandlers` 暴露 `/uploads/**` 静态资源

#### 6. 📐 架构原则

- **前后端分离**：前端 SPA + 后端 RESTful（Spring Boot Controller）
- **无状态认证**：JWT，token 失效靠 expire 时间，后端不维护 session
- **关注点分离**：Pinia 管数据状态 / Vue Router 管路由 / 拦截器管 token 注入；后端 Controller-Service-Mapper 三层
- **降级优先**：所有外部依赖（AI 调用、可选弹窗）失败时业务继续，给用户友好提示
- **历史不可丢**：练习日志删除走严格控制，外键 RESTRICT 保护聚合统计的正确性
